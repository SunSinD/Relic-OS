# Graphics And UI

TempleOS is strongly tied to immediate visual interaction. Relic OS should treat
graphics and UI as core system behavior, not an afterthought.

## Topics

- Display mode.
- Framebuffer ownership.
- Drawing primitives.
- Text rendering.
- Window or pane model if one exists.
- Input focus.
- Mouse and keyboard handling.
- UI compatibility with original programs.

## Visual Direction

The project mark in `docs/assets/relic-os-mark.svg` is a repository-native mark
inspired by the cyan-on-black relic image shared during project setup. Keep
branding assets simple, inspectable, and replaceable.
