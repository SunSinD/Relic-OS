# Architecture Decision Records

ADRs record decisions that are expensive to reverse or important to understand
later.

## When To Write An ADR

Write an ADR when a change affects:

- Boot strategy.
- CPU architecture support.
- Programming language or compiler choices.
- Kernel boundaries.
- Compatibility promises.
- Filesystem formats.
- Build system.
- Repository structure.
- Security model.

## Format

Copy `0000-adr-template.md`, give it the next number, and keep it concise.
