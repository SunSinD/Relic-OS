# Specs

Use this directory for documents that define expected behavior more formally
than normal notes.

Examples:

- Boot protocol contract.
- Filesystem format description.
- Program loading ABI.
- Driver interface.
- Compatibility test format.

A spec should explain:

- Status.
- Scope.
- Inputs.
- Outputs.
- Error behavior.
- Compatibility requirements.
- Tests that prove it works.
