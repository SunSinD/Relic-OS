# Tools

Build, analysis, migration, and developer scripts belong here.

Tooling rules:

- Prefer scripts that run from the repository root.
- Document required host dependencies.
- Print commands and versions when useful for debugging.
- Avoid hidden machine-specific assumptions.
- Keep destructive actions behind explicit flags.

Suggested future tools:

- Source inventory generator.
- Disk image builder.
- Emulator launcher.
- Documentation link checker.
- Compatibility test runner.
