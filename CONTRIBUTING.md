# Contributing

Keep contributions small, clear, and easy to review.

## Good First Work

- Improve a short doc page.
- Add TempleOS source notes under `research/`.
- Add or clean up a build/tooling script.
- Add tests once there is code to test.
- Document a known limitation.

## Pull Requests

- Make one focused change at a time.
- Explain what changed and why.
- Keep formatting noise out of unrelated files.
- List how you checked the change.
- Include license/source notes when adding external code or assets.
